  # Contributing to Focus Adventure

Thank you for your interest in contributing to Focus Adventure.

Focus Adventure is an open-source project built with children with ADHD in mind. Our mission is to create fun, accessible, engaging activities that support the practice of focus-related and cognitive skills through games and interactive experiences.

Our commitment is to keep the core Focus Adventure experience free and accessible to children and families.

We welcome developers, designers, testers, educators, parents, accessibility specialists, researchers, students, translators, and first-time open-source contributors.

You do not need to be an expert to contribute.

Every meaningful contribution matters.

---

## Ways You Can Contribute

There are many ways to help Focus Adventure.

### Developers

You can help with:

- New cognitive mini-games
- Focus and attention activities
- Memory games
- Reaction and inhibition games
- Difficulty and progression systems
- Quests and achievements
- Performance improvements
- Mobile responsiveness
- PWA and offline functionality
- Bug fixes
- Automated testing
- Code quality and refactoring

### UI/UX Designers

You can help improve:

- Child-friendly interfaces
- Game interactions
- Animations
- Navigation
- Visual feedback
- Accessibility
- Progress visualization
- Onboarding
- Responsive design

Please remember that the intended audience includes children. Interfaces should be clear, encouraging, accessible, and avoid unnecessary complexity or manipulative design patterns.

### ADHD, Education, and Research Contributors

We welcome responsible input from:

- ADHD specialists
- Psychologists
- Researchers
- Educators
- Special education professionals
- Occupational therapists
- Parents and caregivers

Research references, evidence-informed suggestions, and reviews of game concepts are valuable contributions.

Please do not present unsupported claims as established scientific or medical facts.

### Accessibility Contributors

We especially welcome improvements involving:

- Keyboard accessibility
- Screen-reader support
- ARIA implementation
- Color contrast
- Reduced-motion support
- Touch target sizing
- Cognitive accessibility
- Clear instructions
- Alternatives to color-only feedback

### Testers

You can help identify:

- Bugs
- Browser compatibility problems
- Mobile layout issues
- Accessibility problems
- Game logic errors
- Scoring problems
- Performance issues
- PWA/offline problems

### Translators and Writers

You can contribute:

- Translations
- Documentation
- Game instructions
- Child-friendly wording
- Developer documentation
- Accessibility documentation
