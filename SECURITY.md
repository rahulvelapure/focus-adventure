  # Security Policy

## Reporting a Security Vulnerability

The security and privacy of Focus Adventure users are important to us, especially because the project is designed with children in mind.

Please do not publicly report security vulnerabilities through GitHub Issues, Discussions, or other public channels.

If you discover a potential security vulnerability, please report it privately using GitHub's **Private Vulnerability Reporting** feature.

Please include, where possible:

- A description of the vulnerability
- Steps to reproduce the issue
- The potential impact
- Affected components or files
- Any suggested remediation

Please avoid accessing, modifying, downloading, or exposing data that does not belong to you while investigating a vulnerability.

## Responsible Disclosure

We ask security researchers to give project maintainers reasonable time to investigate and address reported vulnerabilities before publicly disclosing them.

We appreciate responsible security research that helps make Focus Adventure safer for children, families, and the wider community.

## Security Considerations for Contributors

Contributors must never commit:

- Passwords
- API keys
- Access tokens
- Private credentials
- Secrets from `.env` files
- Private certificates or cryptographic keys
- Personal or sensitive user information

Features involving authentication, analytics, cloud storage, children's data, third-party integrations, or external APIs should receive appropriate security and privacy review before being merged.

## Supported Versions

Focus Adventure is currently under active development.

Security updates are primarily applied to the latest version of the project on the maintained branch.

| Version | Supported |
| --- | --- |
| Latest | Yes |
| Older versions or forks | No |

## Scope

Security reports may include issues involving:

- Authentication or authorization
- Exposure of sensitive information
- Cross-site scripting (XSS)
- Injection vulnerabilities
- Dependency vulnerabilities with a demonstrated impact
- Insecure storage or transmission of information
- Privacy vulnerabilities
- Vulnerabilities involving children's data
- Other issues that could meaningfully compromise users or the application

Thank you for helping keep Focus Adventure safe.
